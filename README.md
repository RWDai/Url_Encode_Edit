# URL Encode Edit

一个可以直接托管在 GitHub Pages 上的纯前端小工具，用来实时查看和编辑 URL 编码字符串。用户在页面顶部输入编码内容后，应用会自动：

- 解码字符串并拆分为**最小单元（逐个字符/表情）**；
- 为每个单元渲染独立的输入框，允许单独修改；
- 将修改后的内容即时重新编码并同步到原始输入框，实现双向绑定。

## 功能亮点

- 💡 实时解析：键入即解码，错误输入会显示友好提示。
- ✏️ 最小单元编辑：支持多语言字符、Emoji，每个单元都是独立输入框。
- ♻️ 双向绑定：修改任意单元，编码字符串立即更新；反向修改编码也会刷新单元。
- 🌓 浅/深色自适应：跟随系统配色，默认即可发布在 GitHub Pages。

## 快速开始

1. 克隆或下载本仓库。
2. 直接双击 `index.html`，即可在浏览器中离线使用。
3. 若需本地服务（便于手机调试），可运行：

   ```bash
   npx serve .
   ```

## 部署到 GitHub Pages

1. 推送到 GitHub 仓库。
2. 打开仓库的 **Settings → Pages**，将 **Build and deployment** 设置为 `Deploy from a branch`，并选择分支（通常是 `main`）和根目录 `/`。
3. 保存后等待 GitHub Pages 构建完成，即可通过分配的 URL 访问该工具。

## 文件结构

- `index.html`：页面结构与可访问性语义标签。
- `styles.css`：响应式布局、浅/深色主题、Token 样式。
- `main.js`：核心逻辑——解码、单元渲染、键盘操作与双向绑定。

## 技术细节

- 使用 `decodeURIComponent` 将输入解析为 Unicode 字符数组，兼容 Emoji。
- 每个字符由独立 `<input>` 管理，`Array.from` 负责按码点切分，保证 surrogate pair 正确。
- 当单元发生变化时，应用会重新拼接并通过 `encodeURIComponent` 生成最新的 URL 编码字符串。
- 提供键盘辅助（左右方向键在单元间跳转）和“新增空白单元”按钮，方便快速编辑。

欢迎根据需要拓展，例如支持 `+` 解码为空格、导出分享等。
